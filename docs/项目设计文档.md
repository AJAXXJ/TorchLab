# TorchLab 项目设计文档

> PyTorch 可视化模型构建平台 — ComfyUI 风格节点蓝图
> 版本：v0.5 · 更新：2026-05-04

---

## 一、产品愿景

打造面向深度学习研究者与工程师的 **Web 端可视化 PyTorch 模型构建平台**，以"节点蓝图"为核心交互范式（对标 ComfyUI / UE5 Blueprint），让用户通过拖拽、连线、配参的方式构建神经网络，自动生成 `nn.Module` 代码。

当前阶段：**Phase 1 进行中** — 多输入/多输出架构已完成，内置节点扩展到 30 种，UE5 蓝图风格主题已应用。

### 2.3 实时 Tensor Shape 推断

- 点击 "Generate Code" 触发拓扑顺序 Shape 推断
- 每个节点实现 `infer_output_shape(input_shapes, params) -> list[tuple]`
- 下游 shape 级联计算，不兼容时返回错误
- 动态 shape（如 RNN 序列长度）使用符号维度 `[B, T, H]` 表示

### 2.4 PyTorch 代码生成

- 点击 "Generate Code"，生成完整 `nn.Module` 子类代码
- 生成内容包括：
  - `__init__()` 中各层实例化
  - `forward()` 按拓扑顺序排列的调用链
  - 中间变量自动命名（`x_{node_id}` 格式）
- 支持 Skip Connection / 多路分叉的变量缓存逻辑
- 错误节点在画布上高亮显示

### 2.5 训练流程管理（Phase 2）

- 训练配置面板：Optimizer / Scheduler / Loss / Epoch / Batch size
- WebSocket 训练指标实时推送
- 支持暂停 / 继续 / 中止
- Checkpoint 管理

### 2.6 模型调试与可视化（Phase 3）

- **激活值监控**：选中节点查看激活值分布直方图
- **梯度流可视化**：反向传播后连线着色表示梯度量级
- **权重分布**：弹窗查看任意层权重/偏置直方图
- **前向 Dry-Run**：随机数据验证整图无误

## 四、核心技术难点与解决方案

### 4.1 Tensor Shape 自动推断

**方案：**
1. 每类节点实现 `infer_output_shape(input_shapes, params) -> output_shape` 抽象方法
2. 按拓扑顺序执行推断（Kahn 算法保证 DAG 顺序）
3. 分支汇合节点（Add/Concat）执行广播兼容性检查
4. 动态维度使用符号变量 `S`（序列长度）占位

```python
# Conv2d shape 推断
def infer_conv2d(input_shape, params):
    B, C_in, H, W = input_shape
    H_out = (H + 2*params['padding'] - params['kernel_size']) // params['stride'] + 1
    W_out = (W + 2*params['padding'] - params['kernel_size']) // params['stride'] + 1
    return [B, params['out_channels'], H_out, W_out]
```

### 4.2 PyTorch forward() 代码生成

**方案：**
1. 拓扑排序确定 `forward()` 执行顺序
2. 为每个节点分配唯一变量名（`x_{node_id}`）
3. Skip Connection：追踪被多个后继引用的节点输出，提前赋值变量缓存

```python
# ResBlock 生成示例
def forward(self, x_1):
    x_2 = self.conv1(x_1)
    x_3 = self.bn1(x_2)
    x_4 = self.relu(x_3)
    x_5 = self.conv2(x_4)
    x_6 = self.bn2(x_5)
    x_7 = x_1 + x_6          # skip connection
    return F.relu(x_7)
```

### 4.3 前后端节点自动发现

**方案：**
1. 后端 `@register_node` 装饰器自动收集所有 ObjectNode 子类
2. `GET /api/nodes` 返回完整元数据（类型、参数模式、端口定义）
3. 前端 `litegraph-factory.ts` 遍历元数据，动态创建 LiteGraph 节点类
4. 新增自定义节点后，前端重新 fetch 即可刷新面板

### 4.4 自定义模块安全

**方案：**
1. 用户必须继承 `ObjectNode` 并使用 `@register_node` 装饰器
2. 静态 AST 分析：禁止 `exec` `eval` `open` `__import__` 等危险调用
3. 禁止导入 `os` `sys` `subprocess` `socket` 等系统模块
4. 运行时通过 `importlib.util` 动态加载

### 4.5 LiteGraph 内置行为覆盖

**问题：** LiteGraph.js 内置右键菜单使用 `position: absolute` 相对画布容器定位，在悬浮侧边栏布局中菜单位置错乱。内置 widget 内联编辑器（`canvas.prompt`）同样存在定位问题，且与统一的 Inspector 编辑范式冲突。

**方案：**
1. **源码级禁用 `LiteGraph.ContextMenu`**：用空壳类替换整个构造函数，20+ 处调用路径全部失效，从根本上消除定位错乱
2. **`pointerdown` 阶段拦截右键**：在 `pointerdown`（先于 `contextmenu` 事件）检测 `button === 2`，调用 `graph.getNodeOnPos()` 定位节点，`preventDefault + stopPropagation` 阻止内置行为
3. **Vue Teleport + `position: fixed` 自定义菜单**：Teleport 到 `<body>`，`clientX/Y` 精确定位，viewport 边界碰撞检测
4. **`canvas.prompt` 覆盖为空函数**：禁用 LiteGraph widget 内联编辑弹窗（`.graphdialog` 元素 `position: absolute` 定位错误），参数编辑统一走 Inspector Vue 表单
5. **`canvas.showSearchBox` 覆盖为空函数**：禁用双击空白搜索

**关键细节：** 右键选中节点用普通 JS 变量（`let ctxNode`）存储，禁止用 Vue `ref()` —— Vue 3 深度 reactive 代理会破坏 LiteGraph 内部 `===` 引用比较，导致 `graph.remove(node)` 静默失败。

### 4.6 Canvas 渲染性能

**方案：**
1. LiteGraph.js 内置虚拟化渲染
2. CSS `radial-gradient` 绘制点阵背景（无 JS 绘制开销，零闪烁）
3. `canvas.clear_background = false` 避免 JS 清空覆盖 CSS 背景

## 八、风险与应对

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| Shape 推断不完整导致用户频繁报错 | 高 | Phase 1 仅支持线性拓扑，复杂分支逐步迭代 |

| 大图（100+节点）画布卡顿 | 中 | LiteGraph.js 虚拟化渲染 |
| 自定义模块安全问题 | 中 | 静态 AST 分析 + importlib 受限加载 |
