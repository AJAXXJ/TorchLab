> 版本：v0.6-dev · 更新：2026-05-06

---

## 一、项目概述

PyTorch 可视化模型构建平台，以 UE5 蓝图风格"节点蓝图"为核心交互范式，拖拽、连线、配参构建神经网络，自动生成 `nn.Module` 代码。

当前阶段：**Phase 1 进行中** — 函数式 Lambda 节点已完成，UE5 蓝图风格组合节点重构中。

## 二、已完成

### 2.1 后端 (FastAPI, port 4003)

| 模块 | 路径 | 说明 |
|------|------|------|
| ObjectNode 抽象基类 | `nodes/object_node.py` | PortDef + ObjectNodeMeta + ObjectNode ABC |
| 全局注册表 | `nodes/registry.py` | @register_node 装饰器 + _NODE_REGISTRY |
| 图构建器 | `core/graph_builder.py` | JSON → DAG + Kahn 拓扑排序 + 多输入边解析 |
| Shape 推断引擎 | `core/shape_inferrer.py` | 拓扑逐节点推断，支持多输出 |
| 代码生成器 | `core/code_generator.py` | nn.Module 代码生成，多输出元组解包，多行 init 缩进 |
| 自定义节点注册 | `controllers/node_controller.py` | AST 安全检查 + importlib 动态加载 |
| 组合节点控制器 | `controllers/composite_controller.py` | 子图封装、持久化、启动加载 |
| 统一响应封装 | `api/response.py` | {msg, code, data} + 业务状态码 |

**API 端点：**

| 方法 | 端点 | 用途 |
|------|------|------|
| `GET` | `/api/nodes` | 返回所有节点元数据（内置 + 自定义 + 组合） |
| `POST` | `/api/generate` | Shape 推断 + PyTorch 代码生成 |
| `POST` | `/api/custom/register` | 上传 .py 代码注册自定义节点 |
| `DELETE` | `/api/custom/{name}` | 删除自定义节点 |
| `POST` | `/api/composite/create` | 从子图创建组合节点类型 |
| `DELETE` | `/api/composite/{name}` | 删除组合节点 |

### 2.2 内置节点（32 种）

**数据节点 (2)：** Input, Output

**函数式节点 (1)：**

| 节点 | 文件 | 说明 |
|------|------|------|
| Lambda | `nodes/layers/lambda_node.py` | 内联 Python 表达式 — 4输入(x0-x3)+3输出(逗号元组)，bracket-aware 输出元数检测 |

**线性层 (1)：** Linear

**卷积层 (3)：** Conv1d, Conv2d, Conv3d

**池化层 (3)：** MaxPool2d, AvgPool2d, AdaptiveAvgPool2d

**归一化层 (5)：** BatchNorm1d, BatchNorm2d, LayerNorm, GroupNorm, RMSNorm

**激活函数 (6)：** ReLU, Sigmoid, GELU, SiLU, Tanh, Softmax

**Dropout (2)：** Dropout, AlphaDropout

**控制流 / 维度操作 (7)：** Add, Concat, Split, Reshape, Permute, Squeeze, Unsqueeze

**嵌入层 (1)：** Embedding

**组合节点基类 (1)：**

| 节点 | 文件 | 说明 |
|------|------|------|
| CompositeNode | `nodes/composite_node.py` | 子图组合基类，动态创建子类注册。子图拓扑排序展开代码，`c_{prefix}` 作用域变量隔离 |

## 四、设计中：UE5 蓝图风格组合节点重构

### 4.1 背景

原实现为"主画布选中节点 → 右键 Group into Node"，用户不满意此交互。
决定改为 **UE5 蓝图 "Collapse to Function" 风格**：

> 用户在独立子画布中编辑组合节点内部逻辑，边界由 Input/Output 节点定义。

### 4.2 方案要点

1. **删除** LiteGraphCanvas.vue 中的：
   - 右键框选逻辑（pointermove/pointerup window 监听器、selectionRect、rightDragging）
   - "Group into Node" 右键菜单按钮及对话框
   - 框选矩形 CSS

2. **启用** LiteGraph 内置 LGraphGroup 作为**视觉注释框**（仅视觉，不影响代码生成）

3. **新建** `CompositeEditorModal.vue`：
   - 独立 LGraph/LGraphCanvas 实例
   - 预置 Input / Output 边界节点（可编辑端口数量）
   - 名称输入 + 保存按钮
   - 调用 POST /api/composite/create 保存

4. **Input/Output 边界节点改造**：支持在 Inspector 中编辑端口数量（num_inputs / num_outputs）

5. **后端不变**：composite_controller.py、composite_node.py 保持现有逻辑
