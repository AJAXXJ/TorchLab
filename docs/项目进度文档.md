# TorchLab 项目进度文档

> 版本：v0.5 · 更新：2026-05-04

---

## 一、项目概述

PyTorch 可视化模型构建平台，以 ComfyUI 风格"节点蓝图"为核心交互范式，拖拽、连线、配参构建神经网络，自动生成 `nn.Module` 代码。

当前阶段：**Phase 1 进行中** — 多输入/多输出架构已完成，内置节点扩展到 30 种，UE5 蓝图风格主题已应用。

---

## 二、已完成

### 2.1 后端 (FastAPI, port 4003)

| 模块 | 路径 | 说明 |
|------|------|------|
| ObjectNode 抽象基类 | `nodes/object_node.py` | PortDef + ObjectNodeMeta + ObjectNode ABC，`to_pytorch_forward` 签名 `output_var: str` → `output_vars: list[str]` |
| 全局注册表 | `nodes/registry.py` | @register_node 装饰器 + NODE_REGISTRY |
| 图构建器 | `core/graph_builder.py` | JSON → DAG + Kahn 拓扑排序 + 多输入边解析（input_sources 按端口索引排序） |
| Shape 推断引擎 | `core/shape_inferrer.py` | 拓扑逐节点推断，支持多输出（tuple of shapes），按 src_out_idx 解析上游端口 |
| 代码生成器 | `core/code_generator.py` | 多输出节点元组解包 (`x_2_0, x_2_1 = torch.chunk(...)`)，端口索引感知变量解析 |
| 自定义节点注册 | `controllers/node_controller.py` | AST 安全检查 + importlib 动态加载 |
| 统一响应封装 | `api/response.py` | {msg, code, data} + 业务状态码 |
| API 路由层 (V) | `api/routers/` | 薄 HTTP 层，参数提取 + 调 controller |
| 控制器层 (C) | `controllers/` | 业务编排，异常 → 业务状态码 |
| 核心引擎 | `core/` | 纯函数引擎，零 HTTP 依赖 |

**API 端点：**

| 方法 | 端点 | 用途 |
|------|------|------|
| `GET` | `/api/nodes` | 返回所有节点元数据（内置 + 自定义） |
| `POST` | `/api/generate` | Shape 推断 + PyTorch 代码生成 |
| `POST` | `/api/custom/register` | 上传 .py 代码注册自定义节点 |
| `DELETE` | `/api/custom/{name}` | 删除自定义节点 |

**统一响应格式：** `{ msg: string, code: int, data: object | null }`

| 节点 | 文件 | 说明 |

|------|------|------|

**池化层 (3)：**

**归一化层 (5)：**

| 节点 | 文件 | 说明 |
|------|------|------|

| RMSNorm | `nodes/layers/normalization.py` | RMS 归一化 |

**激活函数 (6)：**

| 节点 | 文件 | 说明 |
|------|------|------|
| ReLU | `nodes/layers/activation.py` | 修正线性单元 |
| Sigmoid | `nodes/layers/activation.py` | Sigmoid 函数 |
| GELU | `nodes/layers/activation.py` | 高斯误差线性单元 |
| SiLU | `nodes/layers/activation.py` | Sigmoid 线性单元 (Swish) |
| Tanh | `nodes/layers/activation.py` | 双曲正切 |
| Softmax | `nodes/layers/activation.py` | Softmax（可配置 dim） |

**Dropout (2)：**

| 节点 | 文件 | 说明 |
|------|------|------|
| Dropout | `nodes/layers/dropout.py` | 随机失活 |
| AlphaDropout | `nodes/layers/dropout.py` | 保持均值和方差的 Dropout |

**控制流 / 维度操作 (7)：**

| 节点 | 文件 | 端口 | 说明 |
|------|------|------|------|
| Add | `nodes/layers/control_flow.py` | in=2 out=1 | 逐元素加法（广播）— 残差连接 |
| Concat | `nodes/layers/control_flow.py` | in=2 out=1 | 维度拼接 |
| Split | `nodes/layers/control_flow.py` | in=1 out=2 | 沿维度均分张量 |
| Reshape | `nodes/layers/control_flow.py` | in=1 out=1 | 重塑张量形状，支持 -1 自动推断 |
| Permute | `nodes/layers/control_flow.py` | in=1 out=1 | 重排张量维度顺序 |
| Squeeze | `nodes/layers/control_flow.py` | in=1 out=1 | 删除大小为 1 的维度 |
| Unsqueeze | `nodes/layers/control_flow.py` | in=1 out=1 | 在指定位置插入大小为 1 的维度 |

**嵌入层 (1)：**

| 节点 | 文件 | 端口 | 说明 |
|------|------|------|------|
| Embedding | `nodes/layers/embedding.py` | in=1 out=1 | 整数索引→稠密向量，in dtype=scalar |

### 2.4 技术栈总结

| 层 | 选型 |
|----|------|
| 前端框架 | Vue 3 + Composition API + TypeScript |
| 样式 | Tailwind CSS v4 |
| 画布引擎 | LiteGraph.js（npm 依赖） |
| 构建 | Vite |
| 后端框架 | FastAPI |
| 代码生成 | 字符串模板 |
| 图算法 | 自研 (Kahn 拓扑排序) |
| 自定义节点加载 | importlib + AST 安全检查 |

### 2.5 核心架构决策

| 决策 | 结论 |
|------|------|
| 画布引擎 | LiteGraph.js（内置端口类型系统 + 连线校验） |
| 节点定义 | Python ObjectNode 子类是唯一真相源 |
| 前后端对接 | `/api/nodes` 自动发现 → 前端零硬编码渲染 |
| Pin 校验 | 纯前端（LiteGraph 内置 + 前端拥有全部数据） |
| Shape 推断时机 | 构建期（Generate），非编辑期 |
| 后端架构 | MVC: routers (V) → controllers (C) → core (引擎) + nodes (独立) |
| CSS 框架 | Tailwind CSS v4（替代 ant-design-vue） |
| 侧边栏布局 | 悬浮 overlay + translateX 动画，画布不随面板 resize |
| 右键菜单 | 自定义 Teleport 菜单 + 空壳替换 LiteGraph.ContextMenu |
| 参数编辑 | 统一走 InspectorPanel Vue 表单，禁用 LiteGraph 内联编辑 |
| Inspector | 点击节点/生成代码自动展开，手动可折叠 |
| 多输入/多输出 | `to_pytorch_forward` 签名改为 `output_vars: list[str]`；shape_inferrer 按 `src_out_idx` 解析端口 |
| 主题 | UE5 蓝图风格：投影 + 金色发光 + dtype 着色引脚 + 十字线网格 |

│   └── src/
│       ├── main.ts
│       ├── index.css                # Tailwind 入口 + UE5 主题变量
│       ├── App.vue                  # 全幅画布 + 悬浮侧边栏
│       ├── env.d.ts
│       ├── types/
